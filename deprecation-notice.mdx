<Warning>
  **Kuma is no longer actively maintained.** In July 2022, Kuma was officially superseded by [Rumba](https://github.com/mdn/rumba), a new Django-based backend for MDN Web Docs.
</Warning>

## What happened

Kuma served as the backend platform for MDN Web Docs for many years, powering one of the web's most important documentation resources. However, in July 2022, Mozilla decided to transition to a new platform called Rumba to better meet the evolving needs of MDN and its community.

### Timeline

- **Pre-July 2022**: Kuma actively developed and deployed for MDN production
- **July 2022**: Rumba officially replaces Kuma as MDN's backend platform
- **Present**: Kuma repository maintained for historical reference and legacy instances

## Why the change

The transition from Kuma to Rumba was driven by several factors:

<AccordionGroup>
  <Accordion title="Architectural simplification">
    Rumba represents a reimagining of the MDN backend with a focus on simplicity and maintainability. The new architecture reduces complexity while maintaining the essential features needed for MDN's operations.
  </Accordion>
  
  <Accordion title="Modern development practices">
    Rumba incorporates lessons learned from years of Kuma development, implementing modern best practices and updated dependencies to improve security and developer experience.
  </Accordion>
  
  <Accordion title="Platform evolution">
    As MDN evolved, particularly with the introduction of Yari (the frontend platform), the requirements for the backend changed. Rumba was designed specifically to complement Yari's architecture.
  </Accordion>
</AccordionGroup>

## What this means for users

### For MDN visitors

<Note>
  End users of MDN Web Docs were not affected by this transition. The documentation site at [developer.mozilla.org](https://developer.mozilla.org) continued to operate without interruption.
</Note>

The backend platform change was transparent to documentation readers and consumers of MDN content.

### For MDN contributors

Content contribution workflows may have changed with the new platform. Contributors should refer to the current MDN documentation for up-to-date information on how to contribute.

### For API consumers

If you were using Kuma's APIs for:
- Bookmarks management
- User notifications
- Document metadata
- Authentication flows

You should migrate to Rumba's equivalent APIs. Check the [Rumba repository](https://github.com/mdn/rumba) for current API documentation.

## Legacy Kuma instances

If you're maintaining a legacy Kuma instance, be aware:

<Warning>
  Security updates and bug fixes are no longer being actively developed for Kuma. Consider migrating to a supported platform or at minimum ensure your dependencies are kept up to date.
</Warning>

### Recommended actions

1. **Audit your instance**: Understand what features you're using and what data you have
2. **Plan migration**: Evaluate whether Rumba or another platform meets your needs
3. **Update dependencies**: If you must continue using Kuma, regularly update Python packages and Docker images
4. **Monitor security**: Subscribe to security advisories for Django, PostgreSQL, Redis, and other dependencies

## Migration guidance

### Assessing Rumba

Before migrating from Kuma to Rumba, review:

- **Feature parity**: Does Rumba support the features your instance requires?
- **API compatibility**: Will your integrations work with Rumba's APIs?
- **Data migration**: What data needs to be transferred and how?
- **Infrastructure requirements**: Can your infrastructure support Rumba?

## Technical details

### What changed architecturally

Rumba differs from Kuma in several key ways:

1. **Simplified app structure**: Fewer Django apps with clearer separation of concerns
2. **Updated dependencies**: Modern versions of Django, Python, and other core libraries
3. **Streamlined features**: Focus on essential backend services without legacy wiki features
4. **Improved API design**: RESTful APIs designed for integration with Yari frontend

### Data considerations

If migrating from Kuma:

- **User accounts**: OAuth integration may require re-authentication
- **Bookmarks**: May need to be exported and imported
- **Notifications**: Notification history might not transfer
- **Attachments**: Files may need to be migrated to new storage

### Deployment differences

Kuma and Rumba have different deployment requirements:

| Aspect | Kuma | Rumba |
|--------|------|-------|
| Python version | 3.8+ | Check Rumba docs |
| Django version | 3.2 LTS | Check Rumba docs |
| PostgreSQL | 13.3+ | Check Rumba docs |
| Redis | 5.0.6+ | Check Rumba docs |
| Elasticsearch | Used for search | Check Rumba docs |
