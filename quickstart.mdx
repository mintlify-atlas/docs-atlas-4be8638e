This guide will get you up and running with a local Kuma development instance as quickly as possible.

## Prerequisites

- Docker and Docker Compose installed
- Git installed

## Quick setup

<Steps>
  <Step title="Clone and enter the repository">
    ```bash
    git clone https://github.com/mdn/kuma.git
    cd kuma
    ```
  </Step>

  <Step title="Initialize environment variables">
    ```bash
    cp .env-dist.dev .env
    ```
    **Linux users only**: Edit `.env` and change `#UID=1000` to `UID=1000` (or your actual user ID).
  </Step>

  <Step title="Pull and start containers">
    ```bash
    docker-compose pull
    docker-compose up -d
    ```
    Or use the make shortcut:
    ```bash
    make up
    ```
  </Step>

  <Step title="Access the site">
    Open your browser to [http://localhost.org:8000](http://localhost.org:8000)
  </Step>
</Steps>

That's it! Kuma is now running locally.

## Common tasks

### Create an admin user

To access the Django admin interface:

```bash
docker-compose exec web ./manage.py createsuperuser
```

Then visit [http://localhost.org:8000/admin/login](http://localhost.org:8000/admin/login).

### Access the web container shell

```bash
docker-compose exec web bash
```

Or use the make shortcut:

```bash
make bash
```

### View logs

View logs from the web container:

```bash
docker-compose logs web
```

View logs from all containers continuously:

```bash
docker-compose logs -f
```

### Run database migrations

Inside the web container:

```bash
./manage.py migrate
```

### Run tests

Run the Django test suite:

```bash
make test
```

### Restart containers

Restart specific containers after making changes:

```bash
docker-compose restart web worker
```

### Stop and start containers

Stop all containers:

```bash
docker-compose stop
```

Start them again:

```bash
docker-compose up -d
```

## Make shortcuts

Kuma provides convenient `make` commands for common operations:

- `make up` - Start all containers (`docker-compose up -d`)
- `make bash` - Open a bash shell in the web container
- `make shell_plus` - Open Django shell_plus in the web container
- `make test` - Run the test suite

## Development workflow

<Steps>
  <Step title="Edit files on your host">
    Your local directory is mounted at `/app` in the containers. Changes you make to files are immediately reflected in the running containers.
  </Step>

  <Step title="Restart containers if needed">
    Most changes are picked up automatically, but you may need to restart containers after certain changes:
    ```bash
    docker-compose restart web
    ```
  </Step>

  <Step title="Run commands in containers">
    Execute Django management commands or other operations inside containers:
    ```bash
    docker-compose exec web ./manage.py <command>
    ```
  </Step>
</Steps>

## Troubleshooting

### Permission issues (Linux)

If you encounter file permission errors, make sure you've set the `UID` parameter in your `.env` file:

```bash
UID=1000
```

Then restart the containers:

```bash
docker-compose stop
docker-compose up -d
```

### Containers won't start

View container logs to diagnose issues:

```bash
docker-compose logs
```

### Changes to .env not taking effect

After modifying `.env`, you need to restart containers:

```bash
docker-compose stop
docker-compose up -d
```
